## 摘要
近期的视频类世界模型已使基于像素空间的环境在相机层面上变得可交互：用户可以在模型生成连贯视觉续帧的同时导航视点。然而，它们的动作空间仍然不完整：用户只能移动相机，无法对单个物体进行操作。由于真实世界的交互本质上是以物体为中心的，这类模型更接近被动场景观察者，而非真正可操控的环境。本文提出了 WorldCraft，这一框架将交互式视频世界模型从相机导航扩展到物体级的轨迹动作。给定用户点击与草绘路径，WorldCraft 能够生成未来帧，其中被选中的物体沿着预设的轨迹移动，同时相机继续在场景中导航。WorldCraft 通过基于轨迹的控制流程实现此功能：首先，Normalized World Trajectory（NWT，归一化世界轨迹）在与相机无关的世界坐标系中表示用户绘制的运动，并在当前相机姿态下动态重投影，从而将物体运动与由相机引起的屏幕空间位移区分开来；其次，Spatial-Pathway LoRA（SP-LoRA，空间通路 LoRA）通过模型的空间控制通路注入此世界空间信号，在保留预训练相机控制器的同时增加了物体操控能力；最后，Trajectory-Anchored State Persistence（TASP，轨迹锚定状态保持）将世界轨迹视为持久的空间状态，并在轨迹条件生成后刷新自回归记忆，使得移动的物体在离开相机视野后重新出现时保持更新后的位置。实验结果表明，WorldCraft 能够实现精确的物体控制，在仅相机测试中保持视频世界模型的相机保真度，并在长时间自回归推理中（包括物体离开视野的情况）维持物体状态的一致性。

## Abstract
Recent video-based world models have made pixel-space environments interactive at the camera level: users can navigate viewpoints while the model generates coherent visual continuations. Yet their action spaces remain incomplete: users can move the camera, but cannot act on individual objects. Since real-world interaction is inherently object-centric, such models remain closer to passive scene observers than truly manipulable environments. We present WorldCraft, a framework that expands interactive video world models from camera navigation to object-level trajectory actions. Given a user click and a sketched path, WorldCraft generates future frames in which the selected object follows the prescribed trajectory while the camera continues to navigate the scene. WorldCraft achieves this through a trajectory-centric control pipeline: First, Normalized World Trajectory (NWT) represents user-drawn motion in a camera-invariant world coordinate system and dynamically re-projects it under the current camera pose, separating object motion from camera-induced screen-space displacement; Spatial-Pathway LoRA (SP-LoRA) then injects this world-space signal through the model's spatial-control pathway, adding object manipulation capability while preserving the pretrained camera controller; finally, Trajectory-Anchored State Persistence (TASP) treats the world trajectory as a persistent spatial state and refreshes autoregressive memory after trajectory-conditioned generation, allowing moved objects to reappear at their updated positions after leaving the camera view. Experiments show that WorldCraft enables accurate object control, preserves the video-based world model's camera fidelity under camera-only evaluation, and maintains object state across long autoregressive rollouts with off-camera excursions.

## 1. 核心问题与研究背景

- **研究动机**：  
  当前的交互式视频世界模型（如 *Genie 3*、*WorldPlay* 等）能在像素空间中实现相机层面的交互，即用户可自由操控视角而模型持续生成连贯的画面。然而，这类模型的**动作空间仅限于相机导航**，无法对场景内的单个对象进行直接操作。  
  > 现实中的交互行为往往是以物体为中心（object-centric）的，例如推动物体、移动角色或操纵物品。没有对象级控制的模型仅是“被动观察者”，缺乏“可操作世界”的本质特征。

- **核心问题**：  
  如何让视频世界模型在同时支持相机导航的情况下，具备对象级轨迹操控（object-level trajectory control），实现**相机与对象动作的可组合交互**。

## 2. 方法论与技术核心

WorldCraft 提出一个**轨迹中心的控制框架**（trajectory-centric control pipeline），由三大核心组件组成：

### 2.1 Normalized World Trajectory (NWT) —— 相机无关的世界轨迹表示
- **目的**：解耦物体运动与相机运动（ego-motion）。
- **思路**：用户在屏幕上绘制的轨迹会受到相机移动的影响，因此直接使用像素轨迹会错误叠加相机位移。  
- **方法**：  
  - 将用户绘制的2D轨迹升维到**世界坐标系**的归一化平面；
  - 以首帧相机参数为锚点，不断在当前帧下**动态重投影**该轨迹；
  - 结合深度估计（Depth Anything V2）获得物体深度；
  - 用迭代式锚点更新纠正长序列中的几何漂移。
- **优势**：在相机与物体同时运动时仍能保持一致的世界坐标，支持“相机–物体”可组合控制。

### 2.2 Spatial-Pathway LoRA (SP-LoRA) —— 空间控制通路的轻量适配
- **目的**：在不破坏已有相机控制能力的前提下新增物体运动控制。
- **机制**：  
  - 分析发现相机控制与对象轨迹共享同一空间控制通路（spatial-control pathway）；  
  - 因此仅对该通路应用低秩适配（LoRA）而冻结其他参数；
  - 改变的仅是空间位置映射权重（action encoder 与 ProPE 模块），保留主干语义与注意力结构。
- **效果**：避免全量微调破坏相机行为，实现物体控制与相机控制的共存。

### 2.3 Trajectory-Anchored State Persistence (TASP) —— 轨迹锚定状态保持
- **目的**：维持离开相机视野后的对象状态一致性。
- **原理**：  
  - 将世界轨迹 {q_t} 作为“持久空间信号”，即使对象暂时不可见，它在世界空间的轨迹仍继续注入模型；
  - 在相机重新回看时，物体可出现在轨迹更新后的正确位置；
  - 引入“预退出记忆过滤”（pre-exit memory filtering）：在自回归记忆中屏蔽过时的帧，避免对象被固定在旧位置。

## 3. 实验设计

### 3.1 数据与场景
- **构建的新数据集**：
  - 结合 WISA-80K 与 SpatialVID-HQ 两大视频源；
  - 自动提取 (video, camera, mask, trajectory) 四元组，包括：
    - 相机参数（ViPE估计或提供）；
    - 对象掩码（SAM2）；
    - 多点轨迹（CoTracker）；
    - 物理可行性过滤；
  - 共 27,027 条带注释片段，每段 97 帧（3.2s）。
- **数据特征**：
  - 静态相机/移动物体（47%）；
  - 相机与物体同时运动（23%），为 WorldCraft 特有的组合控制场景。

### 3.2 Benchmark 与评测指标
- **主要测试集**：
  1. **Trajectory Accuracy (TA)**：静态相机环境，评估轨迹精度；
  2. **Camera Fidelity (CF)**：仅相机动作输入，测试相机生成稳定性；
  3. **Composable Set (Camera+Trajectory)**：相机与物体同时运动组合测试。
- **评测指标**：
  - **视觉质量**：PSNR / SSIM / LPIPS / DINO / VBench++；
  - **轨迹误差（TE）**：与 CoTracker GT 比较；
  - **相机误差**：旋转 RPE_rot、平移 RPE_trans、综合 RPE_cam，通过 ViPE 估测。

### 3.3 对比基线
- **轨迹控制类（静态相机）**：DragAnything、Wan-Move。  
- **相机控制类（动态相机）**：Yume、Matrix-Game 2.0、GameCraft、WorldPlay。

## 6. 主要结论与发现

1. **对象控制精确**：在静态相机下，轨迹误差 TE 显著低于 DragAnything 与 Wan-Move。  
2. **相机控制保持稳定**：WorldCraft 在相机测试中保持与 WorldPlay 相当的 RPE 指标，长时序（253 帧）甚至更优。  
3. **组合控制成功**：首次实现**相机与对象同时控制**的可组合、可自回归生成。  
4. **状态持久一致**：通过 TASP 机制，离开视野的对象在回视时位置正确、运动连贯。

## 8. 不足与局限

- **范围限制**：仅支持具备用户输入轨迹的目标；无法自动预测未标注物体的离屏动态。  
- **依赖深度估计**：单目估单位估误差在大旋转或极端视点下可能累积投影误差。

- **泛化挑战**：尽管多源数据覆盖丰富，但真实复杂场景的物理一致性仍待进一步验证。
