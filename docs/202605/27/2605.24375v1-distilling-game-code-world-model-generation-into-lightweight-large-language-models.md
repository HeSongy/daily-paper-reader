## 摘要
大型语言模型（LLMs）在从自然语言生成可执行代码方面表现出强大的能力，这为自动构建供 AI 代理使用的环境开辟了可能性。近期在代码世界模型（Code World Models，CWMs）方面的研究表明，LLMs 能够将游戏规则转换为与蒙特卡罗树搜索（Monte Carlo Tree Search）等求解器兼容的 Python 实现。我们针对游戏场景研究这一问题，在此情境下，生成的环境必须实现规则、合法动作、状态转移、观察以及奖励。我们将这些游戏专属的可执行模型称为游戏代码世界模型（Game Code World Models，GameCWMs）。然而，当前生成代码世界模型的方法依赖于最前沿的大模型及推理阶段的优化循环，限制了其可访问性和可扩展性。本研究探讨是否可以通过后训练将 GameCWM 的生成能力蒸馏到较小的模型中。我们引入了：(1) 一个涵盖完全信息和不完全信息游戏的 30 个游戏的精选数据集，(2) 一个用于评估生成代码在结构和语义游戏属性上的验证框架，(3) 一个结合监督微调（Supervised Fine-Tuning，SFT）与可验证奖励的强化学习（Reinforcement Learning with Verifiable Rewards，RLVR）的后训练流程。我们在 Qwen2.5-3B-Instruct 模型上进行了实验，发现 SFT 可以提升语法正确性，而 RLVR 能增强执行层面对游戏规则的遵循性，从而提高 Qwen 在生成完全信息和不完全信息游戏的有效 GameCWM 方面的能力。总体而言，我们的流程使 Qwen2.5-3B-Instruct 更能够生成有效的 GameCWMs，为从自然语言自动生成环境提供了一条可扩展的路径。

## Abstract
Large Language Models (LLMs) have shown great ability in generating executable code from natural language, opening the possibility of automatically constructing environments for AI agents. Recent work on Code World Models (CWMs) demonstrates that LLMs can translate game rules into Python implementations compatible with solvers like Monte Carlo Tree Search. We study this problem in game settings, where generated environments must implement rules, legal actions, state transitions, observations, and rewards. We refer to these game-specific executable models as Game Code World Models (GameCWMs). However, current approaches to generating code world models rely on frontier models and inference-time refinement loops, limiting accessibility and scalability. This work investigates whether GameCWM generation capabilities can be distilled into smaller models through post-training. We introduce: (1) a curated dataset of 30 games spanning perfect and imperfect information games, (2) a verification framework that evaluates generated code against structural and semantic game properties, and (3) a post-training pipeline combining Supervised Fine-Tuning (SFT) with Reinforcement Learning with Verifiable Rewards (RLVR). We experiment with Qwen2.5-3B-Instruct and find that SFT can increase syntactic correctness, while RLVR can improve execution-level adherence to game rules, thereby improving Qwen's ability to generate valid GameCWMs in both perfect and imperfect information games. Overall, our pipeline makes Qwen2.5-3B-Instruct more capable of generating valid GameCWMs, thereby offering a scalable path toward automatic environment generation from natural language.

### 研究动机
- **背景问题**：大型语言模型（LLM）在从自然语言生成代码方面展现出强大能力，但要生成可执行且结构、语义均正确的「游戏环境代码」——即**游戏代码世界模型（Game Code World Models, GameCWMs）**——依然依赖计算成本高昂的前沿大模型（如 GPT‑4 或 Gemini 系列），并需要推理阶段的多次验证与修复循环。  
- **挑战**：
  1. 现有 CWM / GameCWM 方法高度依赖「推理时计算」（test-time compute），不可扩展。

- **核心问题**：是否可以通过「后训练（post-training）」手段，将 GameCWM 的高质量生成能力**蒸馏到轻量级模型**（如 3B 参数级别）中，从而不再依赖昂贵的大模型与测试循环？

## 二、方法论

### 核心思想
通过构建数据集与可验证奖励框架，对小型 LLM（如 Qwen2.5-3B-Instruct）进行两阶段后训练，使其学会生成语法正确、可执行、符合游戏语义的世界模型代码。

### 方法框架
整体包括三大组成部分：

1. **数据集构建模块**  
   - 基于 30 种游戏（含完全/不完全信息）制作 prompt–code 对。  
   - 每个样本包含：API 规格、游戏文字描述、可用动作列表与一个说明性场景（scenario trace）。
   - 游戏分类：简单 → 中等 → 困难 → OOD（如 Generalized Chess、Quadranto、Hand of War、Converge 等）。

2. **层级验证框架（Hierarchical Verification Framework）**  
   四层验证机制，为模型生成代码计算可分解奖励：
   - **Tier 1 静态分析**：检查语法及必备 API 方法存在性。  
   - **Tier 2 动态验证**（Fuzzing）：随机生成游戏轨迹，检测状态确定性、不可变性、终止状态逻辑、执行崩溃。  
   - **Tier 3 语义规则验证**：将生成代码执行轨迹与 by‑LLM‑生成的少量游戏场景对照验证。  
   - **Tier 4 信息一致性检查**：针对不完全信息游戏验证 resample_history 函数是否维持玩家可观测限制。  

   → 奖励分配采用加权综合：
   \[
   r_i=\sum_{t=1}^{4}w_t S_t
   \]
   其中 \(w_1=0.15, w_2=0.25, w_3=w_4=0.3\)，代表各层权重。

3. **两阶段后训练流程**
   - **阶段一：监督微调（Supervised Fine‑Tuning, SFT）**  
     - 目标：学习代码结构和接口规范。  
     - 损失：标准 next-token 预测。  
     - 实施：使用 LoRA（r=16, α=32）微调。  
   - **阶段二：基于可验证奖励强化学习（Reinforcement Learning with Verifiable Rewards, RLVR）**  
     - 算法：Group Relative Policy Optimization（GRPO）。  
     - 奖励来源：执行层级的多级验证框架输出。  
     - 目标：强化代码的语义正确性与执行合规性。
