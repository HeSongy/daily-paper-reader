## 摘要
在大规模户外环境中运行的自主地面机器人需要既具备坚实的远程导航能力，又具备精细的“最后一公里”探索能力。目前视觉语言导航（VLN）的最新进展在短程任务中表现良好，但在长距离任务中缺乏地理空间的定位能力。一些依赖云端大型语言模型（LLM）的基于OpenStreetMap（OSM）的方法容易产生事实性幻觉，且无法根据人类指令执行“最后一公里”的探索。为应对这些挑战，我们提出了G-DRAGON，这是一种用于户外开放世界导航的检索增强框架。该框架通过基于轻量级LLM的生成式检索，将自然语言命令映射到版本化的本地OSM实体，从而生成用于全球路径规划的精确坐标。一个高层规划模块将全局拓扑路径与SLAM系统相连接，将地理空间航点投影到机器人可导航的坐标框架中。对于“最后一公里”阶段，框架切换为基于前沿的探索和开放集语义体素映射，用以定位开放词汇目标。仿真实验结果表明，该框架在性能上优于当前最先进的基线方法。此外，我们还在未见过的真实城市环境中使用无人地面车辆（UGV）进行了验证，成功完成了最长达500米的人员搜索任务。

## Abstract
Autonomous ground robots operating in large-scale outdoor environments require both robust long-range navigation and fine-grained ''last-mile'' exploration. Current advances in visual-language navigation (VLN) work well at short-range tasks, lacking geospatial grounding for long-distance missions. Some OpenStreetMap (OSM)-based methods relying on cloud-based Large Language Models (LLMs) are prone to factual hallucination and cannot conduct ''last-mile'' exploration based on human instruction. To address these challenges, we present G-DRAGON, a retrieval-augmented framework for outdoor, open-world navigation. This framework maps natural-language commands to versioned, local OSM entities via generative retrieval based on lightweight LLM, yielding accurate coordinates for global route planning. A high-level planning module bridges global topological routes with the SLAM system, projecting geospatial waypoints into the robot's navigable frame. For the ''last mile," the framework transitions to frontier-based exploration and open-set semantic voxel mapping to localize open-vocabulary targets. Experimental results in simulation demonstrate our framework outperforms state-of-the-art baselines. Furthermore, we validate the system in unseen real-world urban environments on an Unmanned Ground Vehicle (UGV), successfully completing person-search missions with trajectories of up to 500m.

## 一、研究背景与核心问题

- **研究动机：**  
  自主地面机器人（UGV）在城市或校园等**大规模开放环境**中执行任务时，既需具备**长距离全局导航能力**，又需在目标附近执行**“最后一公里”语义探索任务**（如寻找特定人物或目标物体）。  
  现有视觉语言导航（VLN）模型在室内或短程任务中表现良好，但在以下方面存在不足：
  - 缺乏地理空间定位（geospatial grounding），导致在户外环境易“迷路”；  
  - 依赖云端大型语言模型（LLM）进行地理查询，带来**延迟、隐私、安全与幻觉问题**；  
  - 无法执行与人类指令紧密相关的语义检索与探索。  

- **研究目标：**  
  构建一个**离线运行、语义与地理空间结合**的导航框架，使机器人能从自然语言命令推理出目标位置、规划路径，并在到达区域后自主感知与搜索目标物体。

## 二、方法论：G-DRAGON 框架

### （1）总体架构  
G-DRAGON（**Geospatial Reasoning and Dynamic Planning for Retrieval-Augmented Outdoor Navigation**）由三个核心模块组成（见 Fig. 2）：  
1. **GeoQA（Geospatial Question Answering）** — 负责从用户自然语言中检索地理实体；  
2. **RAPPER（Retrieval-Augmented Planner with Programmatic and Executable Reasoning）** — 负责高层任务理解与可执行规划；  
3. **NæVIS（Navigation and Exploration System）** — 负责融合多模态传感器进行长距离导航与“最后一公里”语义探索。

### （2）核心思想  
> 将自然语言命令 → 生成式检索 → 地理空间坐标 → 路径规划 → 动态自主执行  
整个闭环完全在本地执行，实现**离线可解释导航决策**。

### （3）关键算法与技术设计  

#### a. GeoQA：生成式地理检索
- **任务定义：** 从 OSM 数据库 \(K\) 中选出与查询 \(q\) 匹配的唯一实体 \(e^*\)，获得地理坐标。  
- **技术实现：**
  - 为每个 OSM 实体构造分层唯一 ID（如 *“Teaching Area-Building A”*）；  
  - 基于 LLM（Qwen 2.5-14B）执行**两阶段推理**：
    1. *自由推理阶段*：推断语义类别与上下文；
    2. *受限解码阶段*：使用前缀树（Trie）限制生成输出仅来自有效实体集合；
  - 保证输出实体始终合法，避免语义幻觉。  

#### b. RAPPER：层次化任务规划器
- 将自然语言任务解析为**行为树（Behavior Tree, BT）**结构化代码；节点包含：
  - `GlobalPlanning` （全局路径计算）  
  - `FollowPath` （路径跟随）  
  - `Exploration` （语义搜索）  
- 算法流程（Algorithm 1 概要）：
  1. LLM 解析任务 → 生成 BT；
  2. 检索目标坐标 → 生成全局路径；
  3. 执行节点并实时反馈状态；
  4. 在失败节点触发重规划机制；
  5. 到达目标区域后转入 NæVIS 模块；
- **行为树生成格式：** LLM 通过 XML 构造，自动适配机器人动作库的可执行结构。  

#### c. NæVIS：导航与探索系统
- **多模态传感器融合：** RGB-D 相机、LiDAR、IMU；基于 LiDAR-Inertial Odometry（LIO）进行位姿估计。  
- **路径坐标变换公式：**  
  根据初始 5m 直行轨迹推断地理方位角 \(θ_{bias} = atan2(Δy_{ENU}, Δx_{ENU})\)，将全局航点投影至局部参考系：
  \[
  W^{local}_i = R(θ_{bias})^T (W^{global}_i - p_{end})
  \]
- **非均匀采样策略：** 根据路径曲率 \(\kappa_i\) 动态设定采样间距（直线段 20m，转角段 3m）。  
- **最后一公里探索：**
  - 初步 360° 扫描生成语义体素地图；
  - 基于前沿（frontier）策略与点在多边形（PiP）约束筛选探索方向；
  - 查询条件下语义分割 + DBSCAN 聚类确定目标中心；
  - 自动切换至目标导航模式。

## 六、主要结论与发现

- **GeoQA 模块：** 在 Easy 集合上超越 GPT‑4o（R@1 = 97.6%），在 Hard 集合上保持较高离线性能。  
- **导航性能：**  
  - 仿真任务中 300m 路径成功率达 100%，800m 仍达 95.6%；  
  - “最后一公里”探索任务成功率高达 62.2%，明显优于 OPEN 与 NoMaD。  
- **真实场景验证：** 4 个任务全部完成，路径效率 SPL 在 83–87 之间，“无 RAPPER” 的对照组则多次失败。  
- **综合性能提升：** 比基线平均提升约 50%，展示出在长程导航和开放语义搜索的鲁棒性。

## 七、优点与亮点

- **离线自治：** 不依赖云端 LLM，可在无网络环境中完全运行。  
- **生成式受限检索：** Trie 约束 + CoT 推理同时消除幻觉与保证输出合法。  
- **结构化规划：** 使用行为树（BT）与 XML 自动生成保障任务可执行、可解释。  
- **多模态融合：** 同步利用 LiDAR、IMU、RGB-D 构建语义体素图，实现对复杂户外环境的稳定建图与探索。  
- **模块化集成度高：** 三层体系（GeoQA–RAPPER–NæVIS）清晰解耦，便于维护与拓展。  
- **真实验证充分：** 从仿真到真实城市部署均获得一致结果。
