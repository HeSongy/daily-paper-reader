## 摘要
模型上下文协议（Model Context Protocol，MCP）统一了大型语言模型（LLM）与外部工具之间的接口，但在智能体如何理解其所处环境方面仍存在根本性的差距。当前的研究范式存在二元分化：任务级规划通常忽视执行阶段的动态性，而反应式执行又缺乏长远的预见能力。我们提出了 MCP-Cosmos，这一框架将生成式世界模型（World Model，WM）融入 MCP 生态系统，以实现预测型任务自动化。通过整合 MCP、世界模型和智能体三种不同技术，我们展示了一种“自带世界模型”（Bring Your Own World Model，BYOWM）策略，使智能体能够在执行前于潜空间中模拟状态转移并优化计划。我们采用 ReAct 与 SPIRAL 两种策略、结合两个规划模型与三个具有代表性的世界模型，在 20 余个 MCP-Bench 任务上进行了实验。结果显示，智能体在环境交互关键指标（如工具成功率与工具参数准确率）方面均有提升。此外，该框架还提出了“执行质量”（Execution Quality）等新的评价指标，从而为理解世界模型相较基线模型的有效性提供了新的洞见。

## Abstract
The Model Context Protocol (MCP) has unified the interface between Large Language Models (LLMs) and external tools, yet a fundamental gap remains in how agents conceptualize the environments within which they operate. Current paradigms are bifurcated: Task-level planning often ignores execution-time dynamics, while reactive execution lacks long-horizon foresight. We present MCP-Cosmos, a framework that infuses generative World Models (WM) into the MCP ecosystem to enable predictive task automation. By unifying three disparate technologies, namely MCP, World Model, and Agent, we demonstrate that a "Bring Your Own World Model" (BYOWM) strategy allows agents to simulate state transitions and refine plans in a latent space before execution. We conducted experiments using two strategies, namely ReAct and SPIRAL with 2 planning models and 3 representative world models over 20+ MCP-Bench tasks. We observed improvements in Agent's environment interaction KPI such as tool success rate and tool parameter accuracy. The framework also offers new metrics such as Execution Quality to generate new insights about the effectiveness of world models compared to baseline.

## 一、研究问题与背景

- **核心问题**  
  当前基于大型语言模型（LLM）和外部工具交互的智能体体系（特别是基于 Model Context Protocol, MCP 的系统）在任务规划与执行之间存在明显脱节。  
  - 传统 **任务级规划** 忽视了执行阶段动态性和环境的随机性。  
  - 反应式框架（如 ReAct）虽能实时响应环境，但缺乏长期规划与前瞻性。  
- **研究动机**  
  作者希望通过将“预测性认知”（Predictive Cognition）引入智能体，使其能够在执行前预测环境变化，减少无效或失败的工具调用，提升任务执行的稳定性与准确度。

## 二、方法论：MCP-Cosmos 框架与核心技术

- **总体思想**  
  MCP-Cosmos 将三种核心技术整合：  
  1. **MCP 协议** —— 作为统一的工具访问接口。  
  2. **世界模型（World Model, WM）** —— 负责在潜空间中模拟状态转移。  
  3. **智能体（Agent）** —— 结合世界模型进行预测性规划与执行。  

- **“自带世界模型”（BYOWM, Bring Your Own World Model）策略**  
  智能体可选择不同类型的世界模型来“预演”任务过程，通过在隐空间推演工具调用效果，优化执行计划。

- **算法结构**  
  论文形式化提出 **WM Infused Planning and Execution Algorithm**：
  - **阶段一：模拟规划（Simulation Phase）**
    - 根据任务指令生成初始状态。
    - 使用世界模型预测每一步工具调用的结果（即模拟观测）。
    - 累积模拟轨迹并选取最优计划。
  - **阶段二：真实执行（Execution Phase）**
    - 在真实 MCP 环境中依次执行选定计划。
    - 若遇到失败，可调用修正机制（可选）。
    - 输出最终结果、执行计划、完整执行轨迹。

  ⮕ 该算法实现了“虚拟试验 + 计划选择 + 实际执行”的闭环体系。

## 三、实验设计

- **数据集与场景选择**
  - 使用 **MCP-Bench** 数据集作为主要评测基准，包含：
    - 28 个 MCP 服务器、257 个跨域工具。
    - 任务按照复杂性分层（基于服务器数量）。
  - 实验选取 **24 个复杂任务**，包括：
    - 6 个三服务器高协同任务。
    - 18 个双服务器跨域任务。

- **对比方法**
  共设计三类智能体架构：
  1. **ReAct（基线）**：无世界模型能力的反应式智能体。  
  2. **ReAct-Plan-Exec**：结合世界模型的前瞻性规划执行框架。  
  3. **SPIRAL-Exec**：基于蒙特卡洛树搜索（MCTS）和 LLM 的深度规划智能体。

- **世界模型类型**
  - **LLM 型**：GPT-OSS-120B、Claude-Sonnet-4.6  
  - **MCP 专用型**：Arctic-AWM-4B（Agent World Model）

- **评估指标**
  - MCP-Bench 原有指标：
    - Task Completion、Tool Selection、Planning Effectiveness  
  - 新增指标：
    - **Execution Quality**：反映工具调用成功率与调用效率（惩罚过多调用、奖励高成功率）。

## 六、主要结论与发现

1. **性能提升**
   - 所有世界模型增强型智能体在“工具选择准确度”、“参数选取准确度”、“执行效率”等维度均优于基线 ReAct。
   - 最优配置：SPIRAL-Exec + GPT-OSS-120B-WM，整体分数达 44.8%（较基线提升约 8.7%）。

2. **新评估指标的作用**
   - Execution Quality 有效区分“成功完成任务但效率低”与“高效完成任务”的智能体。
   - 世界模型提升了调用成功率（如 SPIRAL + Claude 达到 100% 成功率）。

3. **消融研究结论**
   - 更强的规划器（Claude-Sonnet-4.6）虽提高准确率，但显著增加工具调用次数与执行时间。
   - 世界模型能约束强规划器的探索范围，使其更高效。

4. **总的发现**
   - 世界模型帮助智能体在执行前进行预测性模拟，从而优化计划、减少无效调用。
   - 但任务完成率仍略低于基线的反应式执行框架（因为其偏向长期规划而非短期修正）。
