## 摘要
近年来，视觉-语言模型的进展使移动 GUI 智能体能够感知视觉界面并执行用户指令，但在长周期及高风险交互中，可靠地预测动作后果仍然至关重要。现有的移动世界模型仅提供基于文本或基于图像的未来状态，但目前尚不清楚哪种表示更有用、生成的 rollout 是否可以替代真实环境，以及测试阶段的指导如何影响不同能力的智能体。为回答上述问题，我们筛选并标注了移动世界模型的数据集，并在四种模态上训练世界模型：delta 文本、完整文本、基于扩散的图像以及可渲染代码。这些模型在 MobileWorldBench 和 Code2WorldBench 上均取得了最先进（SoTA）性能。此外，通过在 AITZ、AndroidControl 和 AndroidWorld 上评估它们的下游效用，我们得到三项发现。第一，可渲染代码重建在分布内具有较高的保真度，并为数据构建提供有效的多模态监督，而基于文本的反馈在在线分布外（OOD）执行中更为稳健。第二，世界模型生成的轨迹能在训练过程中提供可迁移的交互经验，并提升智能体端到端任务性能，尽管这些数据并不保留原始分布。最后，对于低动作熵且过度自信的移动智能体，后验自我反思带来的提升有限，这表明世界模型更适合作为先验感知或训练监督，而非普适的后验验证器。

## Abstract
Recent advances in vision-language models have enabled mobile GUI agents to perceive visual interfaces and execute user instructions, but reliable prediction of action consequences remains critical for long-horizon and high-risk interactions. Existing mobile world models provide either text-based or image-based future states, yet it remains unclear which representation is useful, whether generated rollouts can replace real environments, and how test-time guidance helps agents of different strengths. To answer the above questions, we filter and annotate mobile world-model data, then train world models across four modalities: delta text, full text, diffusion-based images, and renderable code. These models achieve SoTA performance on both MobileWorldBench and Code2WorldBench. Furthermore, by evaluating their downstream utility on AITZ, AndroidControl, and AndroidWorld, we obtain three findings. First, renderable code reconstruction achieves high in-distribution fidelity and provides effective multimodal supervision for data construction, while text-based feedback is more robust for online out-of-distribution (OOD) execution. Second, world-model-generated trajectories can provide transferable interaction experience in the training process and improve agents' end-to-end task performance, although these data do not preserve the original distribution. Last, for overconfident mobile agents with low action entropy, posterior self-reflection provides limited gains, suggesting that world models are more effective as prior perception or training supervision than as universal post-hoc verifiers.

## 一、核心问题与研究背景

- **研究动机**：移动 GUI 智能体（Mobile GUI Agents）需要在复杂视觉界面中理解用户指令并进行交互。传统基于视觉-语言模型（VLM）的智能体能够执行任务，但在**长周期、高风险或需前瞻预测的场景**下，仍缺乏对未来状态的可靠建模。  
- **研究核心问题**：
  1. 哪种未来状态表示（文本、图像、可渲染代码）对 GUI 智能体最有效？
  2. 世界模型生成的虚拟交互轨迹（rollout）是否可以替代真实环境数据？
  3. 测试阶段的世界模型反馈如何作用于不同能力、不同类型的智能体？
- **研究意义**：通过引入“移动世界模型”（Mobile World Model, MWM），尝试构建能预测 GUI 状态变化的统一模拟机制，以提升感知质量和交互效率，并探讨其在训练与测试中的作用边界。

## 二、方法论与技术路线

- **核心思想**：将智能体的未来状态预测任务形式化为**跨模态世界建模问题**，通过多模态生成模型预测下一步的界面、文本或可执行代码，从而指导智能体决策。
- **四种模态建模方法**：
  1. **Delta-Text 模型**：以 GUI 的文本差分为预测目标，只生成变化部分（节省 token、提升时效性）。
  2. **Full-Text 模型**：完整生成下一屏幕的文本描述。
  3. **Diffusion-Image 模型**：基于扩散模型生成下一步界面的图像，模拟视觉变化。
  4. **Renderable-Code 模型**：生成可在浏览器或移动设备上渲染的 HTML/CSS 代码，用于重构未来状态。
- **核心技术要点**：
  - 数据预处理：筛选与标注大规模移动 GUI 交互数据（AITZ、AndroidControl、AndroidWorld）。  
  - 模型训练：为每种模态建立预测器，输入为当前界面与动作，输出为未来状态的表示。  
  - 多模态监督：在训练中引入跨模态一致性损失，使文本与图像预测相互约束。
  - 测试时引导（Test-time Guidance）：通过世界模型预测结果调整智能体动作分布，例如降低动作熵或辅助自我反思。
- **算法流程（文字版）**：
  1. 输入当前 GUI 状态 \( S_t \) 与动作 \( A_t \)；
  2. 世界模型 \( W_\theta(S_t, A_t) \rightarrow \hat{S}_{t+1} \) 预测下一状态；
  3. 将预测结果反馈给智能体策略网络 \( \pi_\phi \)，用于决策或监督；
  4. 可通过迭代 rollout 生成虚拟轨迹以进行训练强化。

## 六、主要结论与发现

1. **表征效果差异**：
   - 可渲染代码在分布内（In-Distribution）状态重建最精确；
   - 文本模型在分布外（OOD）场景鲁棒性更强；
   - 图像扩散模型生成质量高但易出现动态输入错误。
2. **训练时收益**：
   - 世界模型生成的虚拟轨迹可提升智能体训练性能（AndroidControl 提升显著）。
   - 但生成数据分布与真实环境存在偏差，不可完全替代真实经验。
3. **测试时指导效用**：
   - 世界模型在高熵（不确定动作）区间中有效降低错误率；
   - 对低熵、过度自信的智能体，后验反思几乎无益。
4. **整体结论**：
   - 世界模型主要适合作为**先验感知与训练监督工具**，而非普适的后验验证器。
   - 在多任务 GUI 导航中取得 SoTA 性能。
