## 摘要
视觉-语言-动作（VLA）模型在语义泛化方面表现出色，但往往缺乏对世界动力学的细粒度建模。近期研究探索将视频生成模型作为世界建模的基础，进而提出统一的世界动作模型（World Action Models, WAMs），能够联合建模视觉动态与动作。我们提出 MotuBrain，这是一种在 UniDiffuser 框架下的统一多模态生成模型，采用三流 Transformer 混合架构，能够联合建模视频与动作。单个模型支持多种推理模式，包括策略学习、世界建模、视频生成、反向动力学以及视频-动作联合预测，同时可扩展至异构多模态数据，如仅视频数据和跨执行体机器人数据。为提升实际应用能力，MotuBrain 引入了统一的多视图表示、显式语言-动作耦合以及高效的推理栈，实现了实时部署中超过 50 倍的速度提升。

## Abstract
Vision-Language-Action (VLA) models achieve strong semantic generalization but often lack fine-grained modeling of world dynamics. Recent work explores video generation models as a foundation for world modeling, leading to unified World Action Models (WAMs) that jointly model visual dynamics and actions. We present MotuBrain, a unified multimodal generative model that jointly models video and action under a UniDiffuser formulation with a three-stream Mixture-of-Transformers architecture. A single model supports multiple inference modes, including policy learning, world modeling, video generation, inverse dynamics, and joint video-action prediction, while scaling to heterogeneous multimodal data such as video-only and cross-embodiment robot data. To improve real-world applicability, MotuBrain introduces a unified multiview representation, explicit language-action coupling, and an efficient inference stack, achieving over 50x speedup for real-time deployment.

## 二、方法论与模型架构

### 2.1 核心思想

- **统一多模态生成框架**：基于 UniDiffuser 进行视频与动作的联合建模，使模型在同一生成过程下支持五种推理模式：  
  - 策略建模（Policy modeling）  
  - 世界建模（World modeling）  
  - 视频生成（Video generation）  
  - 反向动力学（Inverse dynamics）  
  - 视频-动作联合预测（Joint video-action prediction）

### 2.2 模型结构

- **三流 Mixture-of-Transformers 架构**：
  - **视频流**：负责预测未来视觉状态（视频潜变量）。
  - **动作流**：生成相应的动作序列。
  - **文本流**：作为条件输入流，参与跨模态注意力但不输出标记，用于加强语言-动作耦合。
- **H-Bridge 注意力设计**：只在中间层采用全模态交互，其余层独立处理视频与动作流，兼顾计算效率与模态独立性。
- **多视图支持**：采用 3D RoPE（旋转位置编码）偏移，使模型能灵活处理任意数量和布局的摄像头输入，无需修改主干结构。

### 2.3 训练与公式概要

- **两阶段预训练：**
  1. 视频分支预训练（Stage 1）：在互联网和多视角视频上训练视频生成能力。
  2. 动作分支预训练（Stage 2）：冻结视频分支，仅更新动作分支，采用**相对末端执行器（EEF）表示**，提升跨平台一致性。  
     - 相对动作定义：  
       \( e_{rel,i} = e_{abs,i} \ominus s = (p_i - p_s, R_s^{-1}R_i, g_i) \)
     - 优势：减少因不同机器人姿态格式而产生的不兼容问题。
- **损失函数**：视频与动作的加权均方误差  
  \( L = λ_v L_v + λ_a L_a \)，其中各分支通过**flow matching**预测速度场。
- **后训练阶段**：
  - 非自回归（Non-AR）：一次性预测整段视频和动作窗口。
  - 自回归（AR）：分块因果掩码，模拟实时执行环境。
  - 使用 **V2A-style 注意力** 实现高效推理，只更新动作流并冻结视频流，减少重复计算。
  
### 2.4 推理与系统优化

- **优化手段（累计速度提升约 50×）**：
  - 噪声采样调整（SNR-based step sampling）
  - torch.compile 编译融合
  - FP8 量化与 GEMM 优化
  - DiT 缓存（DreamZero式时序冗余利用）
  - V2A 异步推理／action-only 前缀
- **执行机制**：分块闭环控制与延迟补偿策略，瓣形融合（smooth decay）减轻动作边界不连续。
