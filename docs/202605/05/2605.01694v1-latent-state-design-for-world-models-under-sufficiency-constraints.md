## 摘要
世界模型对于智能体的重要性仅体现在它所构建的状态之上。该状态必须保留部分信息、丢弃其他信息，并支持某种未来功能：预测、控制、规划、记忆、语义锚定或反事实推理。本文将世界模型研究视为在充分性约束下的潜在状态设计。我们提出了一种功能性分类法，将方法按照其潜在状态的用途分组，而非按架构或应用领域划分：预测性嵌入、递归信念状态、对象/因果结构、潜在动作接口、具身规划接口以及记忆基质。这些角色揭示出基于架构的分类所掩盖的差异，包括预测充分性与控制充分性之间的差距，以及被动视频预测与反事实动作建模之间的差距。该分类法支撑了一种评估框架，根据模型潜在状态旨在满足的充分性约束来评估模型。我们从七个维度比较方法：表征、预测、规划、可控性、因果/反事实支持、记忆和不确定性。所得矩阵可作为诊断工具，以分析潜在状态所保留、丢弃及实现的内容。结论是：可操作的世界模型应当是状态构建方式与任务需求相匹配的模型，而非保留信息量最多的模型。

## Abstract
A world model matters to an agent only through the state it constructs. That state must preserve some information, discard other information, and support some future function: prediction, control, planning, memory, grounding, or counterfactual reasoning. This paper treats world-model research as latent state design under sufficiency constraints.   We propose a functional taxonomy that groups methods by what their latent state is for, rather than by architecture or application domain: predictive embedding, recurrent belief state, object/causal structure, latent action interface, grounded planning interface, and memory substrate. These roles expose distinctions that architecture-based groupings hide, including the gap between predictive sufficiency and control sufficiency, and the gap between passive video prediction and counterfactual action modeling.   The taxonomy supports an evaluation framework that judges a model by the sufficiency constraint its latent state was built to satisfy. We compare methods along seven axes: representation, prediction, planning, controllability, causal/counterfactual support, memory, and uncertainty. We use the resulting matrix as a diagnostic for what a latent state preserves, discards, and enables.   The conclusion that follows is that an actionable world model is the one whose state construction matches the task, not the one that preserves the most information.

## 一、研究动机与问题背景

- **核心问题**：  
  世界模型（World Models, WM）研究长期以来缺乏一个统一的功能框架。不同研究在架构和应用层面各自为政，使“世界模型”一词的外延模糊。  
  作者指出，**世界模型的真正核心在于其“潜在状态（latent state）设计”**：世界模型之所以有价值，在于它如何构造并更新一个足以描述外部世界的内部状态。

- **主要挑战**：  
  以往研究在“模型能预测什么或生成什么”层面进行比较，而忽视了潜在状态对不同任务（预测、控制、规划、记忆、反事实推理）的**信息充分性（sufficiency constraints）**。

- **研究目标**：  
  将“世界模型”统一理解为“在信息压缩与任务充分性之间的权衡问题”，并构建一个基于功能而非架构的研究分类体系，以及相应的评估规范。

## 二、方法论与框架设计

### 1. 基本思想

- 作者提出“**潜在状态设计（Latent State Design）**”的总体框架，将各类世界模型的差异归结为其满足不同功能充分性的方式。
- 研究关注**潜在状态应当保留哪些信息、丢弃哪些信息、支持哪些任务**。
- 世界模型被定义为“**在潜在空间中更新状态以预测外部世界演化的系统**”，要求具备：
  1. 状态（State）：潜在变量能总结超越当前观察的信息；
  2. 时间动态（Temporal Evolution）：能预测未来潜变量或观察；
  3. 动作相关性（Action Relevance）：状态能反映行动的后果。

### 2. 理论支撑与关键命题

提出三条形式化命题（对应潜在状态的充分性关系）：

1. **信念状态包含命题（Belief-state subsumption）**：  
   精确的信念状态可同时满足预测和控制的充分性；
2. **预测充分性 ≠ 控制充分性**：  
   能预测未来观测并不意味着能做出最优决策；
3. **被动预测 ≠ 反事实充分性**：  
   仅基于被动数据学习不足以获得可干预、可推理的系统模型。

这些命题构成了作者的评估基准：每种模型可能在不同充分性维度上失效。

### 3. 潜在状态的六类功能角色（Functional Taxonomy）

| 功能角色 | 主要目标 | 典型代表 |
|-----------|-----------|-----------|
| 预测性嵌入（Predictive Embedding） | 通过表示空间预测未来特征（如 JEPA, LeWorldModel） | I-JEPA、V-JEPA 2 |
| 递归信念状态（Recurrent Belief State） | 强调控制与决策效用（如 Dreamer, MuZero） | DreamerV3、TD-MPC2 |
| 对象/因果结构（Object/Causal Structure） | 显式分解世界为对象与关系，支持因果推理 | FOCUS、Causal-JEPA |
| 潜在动作接口（Latent Action Interface） | 在缺乏显式动作标签时抽象出可操作变化 | AdaWorld、CoLA-World |
| 具身规划接口（Grounded Planning Interface） | 提供价值、语言、几何等可规划的潜在空间 | Grounded WM、Latent-WAM |
| 记忆基质（Memory Substrate） | 在部分可观测环境中保持长期一致性 | RELIC、WorldMem |

### 4. 评价框架与七维指标

提出“**七轴评价矩阵（Seven Evaluation Axes）**”，评价每种模型潜在状态的功能充分性：

1. 表征意义（Representation）  
2. 预测能力（Prediction）  
3. 规划可行性（Planning）  
4. 可控性（Controllability）  
5. 因果 / 反事实性（Causal/Counterfactual）  
6. 记忆一致性（Memory）  
7. 不确定性处理（Uncertainty）

此外提出“**保留–丢弃–启用矩阵**”，强调“信息丢弃”是设计选择而非缺陷。

## 六、主要结论与发现

1. 世界模型研究本质是**潜在状态设计问题**；核心不在于架构，而在于其满足的“功能充分性约束”。  
2. 一个可操作（actionable）的模型，是**在压缩与任务需求之间取得匹配**的模型，而非保留信息最多的模型。  
3. 六种潜在角色构成完整的功能空间；未来研究应根据任务选择适当的“信息压缩目标”。  
4. 提出了七维度评估矩阵与“保留–丢弃–启用”框架，为后续模型分析提供标准化思路。  
5. 制定了五项理论研究议程，用以测试不同充分性约束（如预测→控制、反事实建模、记忆一致性、动作可转移性、生成器闭环验证）。
