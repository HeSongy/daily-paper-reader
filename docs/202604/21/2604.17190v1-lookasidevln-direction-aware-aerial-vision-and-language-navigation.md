## 摘要
空中视觉-语言导航（Aerial VLN）使无人机（UAV）能够遵循自然语言指令，在复杂的城市环境中进行导航。尽管近年来的研究通过大规模记忆图谱和前瞻路径规划取得了进展，但仍受限于浅层的指令理解和较高的计算成本。特别是，现有方法主要依赖地标描述，忽视了方向性线索——这是人类导航中空间语境的重要来源。本文提出了 LookasideVLN，一种新范式，能够利用自然语言中的方向性线索，以实现更准确的空间推理和更高的计算效率。LookasideVLN 包含三个核心组成部分：（1）自我中心的旁视图（Egocentric Lookaside Graph, ELG），动态编码与指令相关的地标及其方向关系；（2）空间地标知识库（Spatial Landmark Knowledge Base, SLKB），通过轻量级记忆检索整合以往的导航经验；（3）旁视多模态大型语言模型导航代理（Lookaside MLLM Navigation Agent），将用户指令、视觉观测以及来自 ELG 的地标-方向信息进行多模态对齐以进行路径规划。大量实验结果表明，LookasideVLN 即使仅采用单层前瞻策略，也显著优于当前最先进的 CityNavAgent，证明利用方向性线索是一种强大且高效的空中视觉-语言导航策略。

## Abstract
Aerial Vision-and-Language Navigation (Aerial VLN) enables unmanned aerial vehicles (UAVs) to follow natural language instructions and navigate complex urban environments. While recent advances have achieved progress through large-scale memory graphs and lookahead path planning, they remain limited by shallow instruction understanding and high computational cost. In particular, existing methods rely primarily on landmark descriptions, overlooking directional cues "a key source of spatial context in human navigation". In this work, we propose LookasideVLN, a new paradigm that exploits directional cues in natural language to achieve both more accurate spatial reasoning and greater computational efficiency. LookasideVLN comprises three core components: (1) an Egocentric Lookaside Graph (ELG) that dynamically encodes instruction-relevant landmarks and their directional relationships, (2) a Spatial Landmark Knowledge Base (SLKB) that provides lightweight memory retrieval from prior navigation experiences, and (3) a Lookaside MLLM Navigation Agent that aligns multimodal information from user instructions, visual observations, and landmark-direction information from ELG for path planning. Extensive experiments show that LookasideVLN significantly outperforms the state-of-the-art CityNavAgent, even with a single-level lookahead, demonstrating that leveraging directional cues is a powerful yet efficient strategy for Aerial VLN.

## 一、研究背景与核心问题

- **研究动机**  
  空中视觉-语言导航（Aerial VLN）任务要求无人机（UAV）依据自然语言指令在复杂城市环境中自主飞行。这类任务在城市场景中面临新的挑战：  
  - **地标歧义性高**：城市中同类地标（如“建筑”“桥梁”“树”等）数量众多，使得基于语义匹配的导航容易出错。  
  - **计算资源开销大**：现有方法依赖大规模全局场景图（global scene graph）进行长序列地标匹配（lookahead path planning），耗时且资源消耗高。  
  - **指令理解浅显**：多数模型忽视语言中的**方向性线索**（如“向左转”“继续直行”“在右侧”），削弱空间推理能力。

- **研究目标**  
  提出一种新的空中导航范式——**LookasideVLN**，通过从**自然语言方向信息**出发，在保持高效率的同时提升无人机的语义感知与路径规划能力。

## 二、方法论与模型设计

### 1. 核心思想
提出“**Lookaside**”导航范式，与以往的“**Lookahead**”方法形成对比。其核心在于利用**自我中心的方向线索（egocentric directional cues）**替代全局路径序列匹配，从而实现**轻量级、高效且语义一致的导航推理**。

### 2. 方法总体结构
LookasideVLN框架由三大模块组成（见论文 Figure 2）：

#### （1）空间地标知识库（SLKB, *Spatial Landmark Knowledge Base*）
- 用于存储以往的导航经验，包括地标描述及三维位置。  
- 基于大语言模型（MLLM）的**地标识别器（Landmark Recognizer）**与**检测器（Detector）**构建。  
- 采用层级结构 `{landmark: [positions]}`，支持快速描述与空间位置联合检索。  
- 更新策略：若新检测地标与已有相似，则合并位置，否则插入新条目。

#### （2）自中心旁观图（ELG, *Egocentric Lookaside Graph*）
- 只关注与当前指令相关的少量地标与其方向关系，不维护全局图。  
- 节点表示未访问的地标候选位置，边表示自我中心方向关系（左、右、前、上、下等）。  
- 方向度量公式：  
  - 计算相邻地标候选距离与偏转角度：  
    \[
    \theta = hangle(p_{i,k}^{i-1,j}, p_{i+1,m}^{i,k}), \quad d = ||(p_{i+1,m} - p_{i,k})_{xy}||_2
    \]  
  - 转换为语言化的指令描述，如“向左15°前进10米上升2米”。  
- 通过遍历ELG的多路径组合，生成若干**候选路径描述文本**供后续选择。

#### （3）旁视多模态语言导航代理（Lookaside MLLM Navigation Agent）
- 以多模态大模型（如 Qwen2.5-VL-72B）为核心，通过链式思考（chain of thought）完成决策。  
- 输入：  
  - 用户指令、ELG生成的候选路径描述、无人机当前全景观测。  
- 输出：  
  - 观察总结 → 导航进度 → 方向路径选择 → 动作推理（选择动作及执行次数）。  
- 支持动作集：前进、左转、右转、上升、下降、停止。

## 七、方法优点与创新点

- **方向线索建模创新**：首次引入“Lookaside”概念，用语言中的方向信息替代复杂全局图推理。  
- **轻量级记忆系统**：SLKB 实现高效回忆与增量更新，减少显存和算力需求。  
- **语言化路径规划**：ELG 转化为文本指令式路径，使 MLLM 可直接进行自然语言推理。  
- **自我中心参考系**：用飞行器视角推理空间方向，贴近人类导航习惯。  
- **强解释性**：通过链式思考输出推理过程，可人工验证导航逻辑。  
- **高效泛化**：无需训练即可在新场景中工作，降低数据标注需求。
