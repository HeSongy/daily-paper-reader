## 摘要
视觉与语言导航（Vision-and-Language Navigation）要求智能体在视觉不断变化的环境中遵循自然语言指令。核心挑战在于语言与视觉观测之间的动态纠缠：随着智能体视野和空间上下文的变化，指令的意义也随之转变。然而，许多现有模型将指令编码为静态的全局表示，限制了它们根据当前视觉上下文调整指令语义的能力。为此，我们将指令理解建模为一个“Instruction-as-State”变量——一个与决策相关的、基于token级的指令状态，它在每一步中根据智能体的感知状态不断演化，其中感知状态指的是每一步中基于观测的导航上下文。为实现这一原则，我们提出了“State-Entangled Environment-Guided Instruction Understanding（S-EGIU）”，一个由粗到细的框架，用于基于状态的片段激活和token级语义细化。在粗粒度层面，S-EGIU激活与当前观测语义对齐的指令片段；在细粒度层面，它通过观测引导的token定位与上下文建模精炼被激活片段，使其在当前观测下语义更加明确。这两个阶段共同维持一个随着智能体感知状态在导航过程中持续更新的指令状态。S-EGIU在多个关键指标上表现出色，包括在REVERIE测试集未见环境上提升了2.68%的SPL，同时在多个VLN基准上表现出一致的效率增益，凸显了动态指令—感知纠缠的价值。

## Abstract
Vision-and-Language Navigation requires agents to follow natural-language instructions in visually changing environments. A central challenge is the dynamic entanglement between language and observations: the meaning of instruction shifts as the agent's field of view and spatial context evolve. However, many existing models encode the instruction as a static global representation, limiting their ability to adapt instruction meaning to the current visual context. We therefore model instruction understanding as an Instruction-as-State variable: a decision-relevant, token-level instruction state that evolves step by step conditioned on the agent's perceptual state, where the perceptual state denotes the observation-grounded navigation context at each step. To realize this principle, we introduce State-Entangled Environment-Guided Instruction Understanding (S-EGIU), a coarse-to-fine framework for state-conditioned segment activation and token-level semantic refinement. At the coarse level, S-EGIU activates the instruction segment whose semantics align with the current observation. At the fine level, it refines the activated segment through observation-guided token grounding and contextual modeling, sharpening its internal semantics under the current observation. Together, these stages maintain an instruction state that is continuously updated according to the agent's perceptual state during navigation. S-EGIU delivers strong performance on several key metrics, including a +2.68% SPL gain on REVERIE Test Unseen, and demonstrates consistent efficiency gains across multiple VLN benchmarks, underscoring the value of dynamic instruction--perception entanglement.

## 一、研究背景与核心问题

- **研究任务**：视觉与语言导航（Vision-and-Language Navigation, VLN）要求智能体在视觉动态变化的三维环境中，根据自然语言指令执行连续行动。  
- **核心挑战**：  
  - 指令含义在导航过程中会随视角与空间上下文改变而演化。  
  - 传统方法常将整句指令编码为**静态全局向量**，难以捕捉这种动态“语言—视觉纠缠”。  
- **研究动机**：  
  - 现有模型无法动态调整指令中不同片段的“当前相关性”，容易出现**指令理解滞后**或**错误激活**。  
  - 因此，作者提出将指令表示为随感知状态动态变化的“指令状态（Instruction-as-State）”，以实现真正的**时序语义对齐与状态条件理解**。

## 二、方法论与算法框架

### 1. 总体思想

- **核心理念**：将自然语言指令建模为一个随环境感知状态演化的动态变量。  
  - 即：“Instruction-as-State” → 指令理解不再是静态编码，而是**逐步演化的决策状态**。
- **总体框架**：提出 **S-EGIU（State-Entangled Environment-Guided Instruction Understanding）**。
  - 两层层级式设计：
    1. **CGIP（Coarse-Grained Instruction Processing）**：依据当前视觉观测激活最相关的指令片段。
    2. **FGIP（Fine-Grained Instruction Processing）**：对激活的片段进行 token 级语义精炼，使语言与视觉语义对齐。

### 2. 核心处理流程（简述）

1. **感知状态建模**  
   - 智能体在时间步 t 拥有全景视觉特征 \(V_t\)，其构成“感知状态”。  
   - 初始指令经文本编码器得到 \(S_0\)，包含每个 token 的语义表示。  

2. **指令状态更新机制**  
   - 在每一步通过函数：
     \[
     S_t = f(S_{t-1}, S_0, I, V_t)
     \]
     表示根据当前视觉状态更新指令语义状态。

3. **CGIP 阶段：子指令激活**
   - 通过两步：
     - **Segment + RefineSeg**：基于规则与可学习边界检测分割出子指令。
     - **Cross-Attention 对齐**：计算与视觉特征的相关性分数，softmax 得到子指令激活分布。
     - 选取最高相关子指令 \(k^*\) 进入下一阶段。
   - 该阶段确保仅激活当前可感知、可执行的片段。

4. **FGIP 阶段：语义细化**
   - 对被激活的子指令 Token 执行 cross-attention，与当前视觉特征进行对齐，获得视觉语义调制后的表示。
   - 通过轻量 Transformer 进行上下文建模。
   - 最终使用门控融合更新：
     \[
     S_t = S_{t-1} + g \odot (R_t - S_{t-1})
     \]
     控制 token 层面的语义更新幅度。

5. **训练策略**
   - **IL + RL 混合训练**：
     - Imitation Learning 提供短期监督、稳定训练；
     - Advantage Actor-Critic (A2C) 进行强化学习优化长期路径回报。
   - 总损失：
     \[
     L = L_{RL} + \lambda L_{IL}
     \]

## 三、实验设计

### 1. 数据集与基准
- **数据集覆盖**三大 VLN 标准基准：

  - **SOON**：长指令与多阶段导航任务，考察长序列语义演化。  
  - **REVERIE**：目标对象导向导航，考察精细 token grounding。

### 2. 对比方法
- **基线模型**：

- **受控对比插件（Controlled plug-ins）**：
  - 在同一 DUET 框架下实现五种机制家族的轻量整合以作公平对比：
    - 记忆增强（MapNav）
    - 探索回溯（E2BA）
    - 检索记忆（VLMaps）
