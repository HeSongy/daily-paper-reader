## 摘要
我们将用于安全控制设计的潜在表示方法扩展到集合值状态。最新研究表明，在学习得到的潜在空间中设计的Barrier函数可以将安全保证传递回原始系统，但这些方法仅在单点状态上评估安全证书，从而忽略了状态的不确定性。固定的安全裕度可以部分缓解这一问题，但无法适应不同安全约束间不确定性间隙的各向异性和时变特性。为此，我们将系统状态表示为Zonotope，通过编码器传播得到潜在Zonotope，并在整个集合的最坏情况下评估安全证书。在一个16维四旋翼悬挂载荷穿越门任务中，基于集合的评估实现了5次中5次无碰撞通过，而基于点的评估仅为1次，固定裕度基线为2次。集合评估在每个head的评估中报告安全的比例为44.4%，而点评估为48.5%；这种更高的保守性检测到了4.1%的盲区，在这些盲区中点评估错误地认证安全，从而实现更早的纠正控制。点评估与集合评估之间的安全差距在不同证书head中变化高达12倍，这解释了为何单一固定裕度无法满足要求，并验证了基于集合评估通过结构性方式实现每个head与每个时间步自适应的必要性。

## Abstract
We extend latent representation methods for safety control design to set-valued states. Recent work has shown that barrier functions designed in a learned latent space can transfer safety guarantees back to the original system, but these methods evaluate certificates at single state points, ignoring state uncertainty. A fixed safety margin can partially address this but cannot adapt to the anisotropic and time-varying nature of the uncertainty gap across different safety constraints. We instead represent the system state as a zonotope, propagate it through the encoder to obtain a latent zonotope, and evaluate certificates over the worst case of the entire set. On a 16-dimensional quadrotor suspended-load gate passage task, set-valued evaluation achieves 5/5 collision-free passages, compared to 1/5 for point-based evaluation and 2/5 for a fixed-margin baseline. Set evaluation reports safety in 44.4% of per-head evaluations versus 48.5% for point-based, and this greater conservatism detects 4.1% blind spots where point evaluation falsely certifies safety, enabling earlier corrective control. The safety gap between point and set evaluation varies up to $12\times$ across certificate heads, explaining why no single fixed margin suffices and confirming the need for per-head, per-timestep adaptation, which set evaluation provides by construction.

## 一、核心问题与研究背景

- **研究动机**：  
  传统的安全控制方法利用控制屏障函数（Control Barrier Function, CBF）在状态空间中构造安全集的前向不变性，以确保系统不发生危险行为。然而，在复杂高维系统中构造CBF公式通常计算量巨大、不具可扩展性。  
  最近的潜空间学习控制方法通过学习一个低维潜表示，使得在潜空间中设计的安全证书可以传递到原系统。然而，既有方法只在**单点状态**上评估安全证书，忽略了传感器噪声和建模误差带来的状态不确定性。

- **核心问题**：  
  如何在潜空间安全控制框架下，实现对整个**状态不确定性集合**（而非单点）的安全验证，从而获得对真实系统更稳健的安全保证。

## 二、方法论：基于集合的潜空间安全验证

### 1. 核心思想

- 将系统状态表示为**Zonotope（凸集合）**，以捕获状态估计不确定性。
- 通过已训练的**编码器（Encoder）**将状态集合传播到潜空间，得到潜空间的Zonotope集合。
- 在潜空间中，对整个集合的最坏情况进行**安全证书（Barrier函数）评估**，保证集合内所有状态皆安全。

### 2. 关键技术与算法流程

#### （1）Zonotope 表示与传播
- 状态集合定义为  
  \( X = \langle c, G_x \rangle \)，其中 \( c \) 为集合中心， \( G_x \) 为生成矩阵。
- 编码器传播：  
  通过线性层与非线性层（tanh）采用**区间线性包络近似**进行集合传播，得到潜空间集合  
  \( Z = E(X) = \langle c_z, G_z \rangle \)。

#### （2）最坏情况安全证书评估
- 线性屏障函数形式：  
  \( h(z) = w^T z + b \)。
- Zonotope的最小值解析计算：  
  \( h_{\min}(Z) = w^T c_z + b - \sum_j |w^T G_{z,j}| \)。
- 若 \( h_{\min}(Z) \geq 0 \)，则保证集合内所有状态安全。

#### （3）控制屏障函数 (CBF) 优化约束
- 控制输入通过二次规划（QP）求解：
  \[
  \min_\mu \|\mu - \mu_{nom}\|^2  
  \]
  约束条件：
  \[
  w_i^T B\mu + L_f^{min,i} + \alpha h_{min,i} \geq \epsilon_{dir,i}
  \]
- 实现最小修改的安全控制输入 \( \mu_{safe} \)。

#### （4）算法整体流程（Algorithm 1）
1. 构造状态Zonotope。  
2. 通过编码器得到潜空间Zonotope。  
3. 对每个 barrier 头进行最坏情况评估。  
4. 解 QP 以得到安全控制输入。  
5. 返回 \( \mu_{safe} \)。

## 三、实验设计

### 1. 系统与任务
- **系统**：16维四旋翼无人机携带悬挂载荷（刚性杆模型，形成3D摆动系统）。
- **任务场景**：无人机携带负载通过矩形门（1.2m × 1.2m）而不碰撞。
- **控制频率**：外层控制50Hz。

### 2. 数据与训练
- 使用10,000组状态-控制-下一状态样本。
- 采样策略：
  - 随机快照（25%）  
  - 轨迹采样（30%）  
  - 近门重点采样（25%）  
  - 安全边界样本（20%）  
- 编码器为三层MLP结构（Dense-Tanh-Dense-Tanh-Dense），输出潜向量维度8维。

### 3. 对比基线
- **POINT**：仅在状态中心点评估安全证书。  
- **PT MARGIN**：在点评估中加入全局固定安全裕度（δ=0.031）。  
- **SET（本文方法）**：在集合上执行最坏情况评估。

### 4. 测试场景与指标
- 五种不同难度场景（GOOD, BAD, HARD, HARD1, HARD2）。  
- 衡量指标：
  - 碰撞率（Gate passage success）  
  - 安全证书评估率（h≥0比例）  
  - 盲区检测率（点评估假安全的比例）。
