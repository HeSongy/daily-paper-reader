## 摘要
在智能机器人系统中，具身智能与嵌入式部署之间的鸿沟仍是一项关键挑战，其中感知、推理与规划必须在计算、存储、能耗和实时执行等严格约束下协同运行。在视觉—语言导航（VLN）任务中，现有方法往往在强推理能力与现实平台上的高效部署之间面临根本性权衡。本文提出了一种可部署的具身视觉—语言导航系统，能够在真实机器人平台上同时实现高效率与稳健的高层次推理。为此，系统被解耦为三个异步模块：用于持续环境感知的实时感知模块、用于空间—语义聚合的记忆整合模块，以及用于高层决策的推理模块。我们逐步构建一个认知记忆图来编码场景信息，并进一步将其分解为子图，以支持基于视觉—语言模型（VLM）的推理。为了进一步提升导航的效率和准确性，我们还利用认知记忆图将探索问题形式化为上下文感知的加权旅行修理者问题（Weighted Traveling Repairman Problem，WTRP），以最小化视点的加权等待时间。大量在仿真和真实机器人平台上的实验表明，该系统在保持资源受限硬件实时性能的同时，相较现有VLN方法显著提升了导航成功率与效率。

## Abstract
Bridging the gap between embodied intelligence and embedded deployment remains a key challenge in intelligent robotic systems, where perception, reasoning, and planning must operate under strict constraints on computation, memory, energy, and real-time execution. In vision-language navigation (VLN), existing approaches often face a fundamental trade-off between strong reasoning capabilities and efficient deployment on real-world platforms. In this paper, we present a deployable embodied VLN system that achieves both high efficiency and robust high-level reasoning on real-world robotic platforms. To achieve this, we decouple the system into three asynchronous modules: a real-time perception module for continuous environment sensing, a memory integration module for spatial-semantic aggregation, and a reasoning module for high-level decision making. We incrementally construct a cognitive memory graph to encode scene information, which is further decomposed into subgraphs to enable reasoning with a vision-language model (VLM). To further improve navigation efficiency and accuracy, we also leverage the cognitive memory graph to formulate the exploration problem as a context-aware Weighted Traveling Repairman Problem (WTRP), which minimizes the weighted waiting time of viewpoints. Extensive experiments in both simulation and real-world robotic platforms demonstrate improved navigation success and efficiency over existing VLN approaches, while maintaining real-time performance on resource-constrained hardware.

## 二、方法论与系统设计  

### 1. 整体架构：分层异步认知系统（HiCo-Nav）
系统被分解为三个**异步工作层**，以实现多尺度认知与实时执行：  
- **层 1：实时感知与执行层**  
  - 综合视觉（RGB-D）、LiDAR 感知、SLAM 定位、避障和路径规划。  
  - 高频运行（约 20 Hz），保证动态响应。  
- **层 2：记忆构建层**  
  - 构建“认知记忆图”（Cognitive Memory Graph, CMG），整合空间和语义信息。  
  - 将视觉锚点（Visual Anchors）和对象节点（Object Nodes）组织为二部图 \( G = (V_a, V_o, E) \)。  
  - 使用 **YOLO-World** 进行开放词汇检测，**Mobile-SAM** 分割，**CLIP** 提取语义特征；通过几何与语义相似度合并节点：  
    \[
    s(o, o') = \lambda_1 s_g(o,o') + \lambda_2 s_s(o,o')
    \]
  - 通过**整数线性规划（ILP）**优化视觉锚点选择，保证记忆的紧凑性与覆盖性。  
- **层 3：高层推理层**  
  - 异步地调用 **VLM（如 Qwen3-Omni）** 对记忆子图进行推理。  
  - 将任务语言指令分解为目标对象集 \( O_t \) 和相关对象集 \( O_r \)，以局部优先级子图进行推理。  
  - 优先得分计算：
    \[
    S(G_k) = \sum_{o_i \in V_k^o} s(o_i, T)
    \]
    其中 \( s(o_i, T) \) 是对象与指令的 CLIP 相似度。  

### 2. 探索与规划策略  

**上下文感知前沿探索（Context-Aware Frontier Exploration）**  
- 将前沿点（frontier）打分为：
  \[
  S(f_i) = \lambda_1 S_{struct}(f_i) + \lambda_2 S_{vis}(f_i)
  \]
- 其中：
  - \( S_{struct} \)：由场景语义传播建模，反映物体间的空间关联。  
  - \( S_{vis} \)：由边界外视觉证据（未探索区边缘的图像）通过 CLIP 相似度计算。  

**全局目标选择**  
- 将探索问题形式化为**加权旅行修理工问题（WTRP）**：
  \[
  \min_{\pi} \sum_i W_{\pi_i} C_i(\pi)
  \]
  其中权重 \( W_{\pi_i} \) 与前沿效用相关。  
- 通过指数重权：
  \[
  W_{\pi_i} = \frac{\exp(\beta \bar{S}(f_i))}{\exp(\beta)}
  \]
- 使用 **LKH 启发式算法**求解最优访问序列，以最小化目标发现时间。  
- 局部规划使用 **A*** 算法与平滑轨迹优化，生成可行路径并实时控制机器人运动。

## 五、主要结论与发现  

- HiCo-Nav 在所有基准上均取得**最优性能**：

## 七、不足与局限  

- **空间维度限制**：系统主要基于 2D 平面地图，无法处理跨楼层导航。  
- **感知误差**：在光照或运动抖动下，小物体检测不稳定。  
- **对外部模型依赖高**：依赖大型预训练的 VLM / LLM，如 Qwen3-Omni，可能带来资源与网络依赖风险。  
- **评估时未提供能耗或计算分析**：文章未详细报告算力消耗或实时帧处理负载的量化结果。  
- **通用性仍有限**：虽然适配多任务，但实验集中于室内场景，户外或多机器人环境尚未验证。
