## 摘要
具身智能体越来越依赖大型语言模型（LLM）进行规划，但逐步调用 LLM 会导致严重的延迟和成本。在本文中，我们表明具身任务表现出显著的规划局部性，即下一个计划在很大程度上可以由当前计划预测。基于这一发现，我们提出了 AgenticCache——一种通过重用缓存计划来避免逐步调用 LLM 的规划框架。在 AgenticCache 中，每个智能体会查询运行时缓存以获取常见的计划转移，而后台的缓存更新器（Cache Updater）则异步调用 LLM 来验证和优化缓存条目。在四个多智能体具身任务基准上，AgenticCache 在12种配置（4个基准 × 3个模型）中平均将任务成功率提高了22%，将仿真延迟降低了65%，并减少了50%的 token 使用量。基于缓存的计划重用因此为实现低延迟、低成本的具身智能体提供了一条可行途径。代码可在 https://github.com/hojoonleokim/MLSys26_AgenticCache 获取。

## Abstract
Embodied AI agents increasingly rely on large language models (LLMs) for planning, yet per-step LLM calls impose severe latency and cost. In this paper, we show that embodied tasks exhibit strong plan locality, where the next plan is largely predictable from the current one. Building on this, we introduce AgenticCache, a planning framework that reuses cached plans to avoid per-step LLM calls. In AgenticCache, each agent queries a runtime cache of frequent plan transitions, while a background Cache Updater asynchronously calls the LLM to validate and refine cached entries. Across four multi-agent embodied benchmarks, AgenticCache improves task success rate by 22% on average across 12 configurations (4 benchmarks x 3 models), reduces simulation latency by 65%, and lowers token usage by 50%. Cache-based plan reuse thus offers a practical path to low-latency, low-cost embodied agents. Code is available at https://github.com/hojoonleokim/MLSys26_AgenticCache.

## 一、研究背景与核心问题

- **研究动机**：  
  随着具身智能体（Embodied AI agents）在复杂任务中越来越多地使用大语言模型（LLM）进行“感知–规划–执行”循环，LLM 的调用次数成为主要瓶颈。每一步都触发一次 LLM 推理，会带来：
  - **高延迟**（实时性差），  
  - **高成本**（token 消耗较大）。  
  尽管已有并行规划（Parallelized Planning-Acting）和推测式规划（Speculative Planning）等方法能部分缓解延迟，但都仍需要在每步进行 LLM 调用。

- **核心问题**：  
  能否利用具身任务中的“**计划局部性（plan locality）**”——即当前计划与下一步计划往往具有可预测性——通过复用和缓存机制减少 LLM 实时调用，从而提升智能体的执行效率与经济性？

- **论文主旨**：  
  提出 **AgenticCache** 框架，通过在具身智能体中加入“缓存式规划器”，实现异步、低成本的计划生成，避免每步依赖 LLM 推理。

## 二、方法论与技术细节

### 1. 核心思想

- 具身任务呈现强计划局部性：  
  例如“go grasp target”之后往往接“put into container”；此规律在多场景中稳定出现。
- **AgenticCache 核心机制**：
  1. **缓存驱动的计划复用**：缓存常见的计划转移模式（Plan-to-Plan transitions），快速查表代替LLM推理。
  2. **异步缓存更新（Cache Updater）**：后台周期性调用 LLM 验证缓存条目，并根据反馈动态修正（确认 / 替换）。
  3. **混合决策机制**：在缓存命中时使用缓存结果；在缓存失效或环境变化时再启用 LLM。

### 2. 缓存结构与算法设计

- **缓存条目形式**：  
  每个智能体维护独立缓存，记录二元组 ⟨Pi → Pj⟩，代表从计划 Pi 转移到 Pj 的模式。  
  每项包含：
  - 转移统计 **C(Pi→Pj)**：出现次数；
  - 重要性 **I(Pj)**：由 LLM 确认的次数比例；
  - 状态元数据（如已完成子任务数、访问房间数、时间步范围等）。

- **评分函数**：  
  ```
  S(Pi → Pj) = C(Pi → Pj) × I(Pj)
  ```
  通过转移频度与全局验证频率的乘积挑选最可信的下一步计划。

- **更新与修正机制**：
  - 若 LLM 异步验证结果匹配当前计划 → 增强化缓存条目；
  - 若不匹配 → 减弱错误项，登记新的计划替换。
  - 同时支持“确认压制”和“修正压制”，即在短时内避免重复无效验证。

- **缓存失效处理**：  
  若无匹配计划（cold miss），临时调用 LLM 获取新计划并更新缓存。
