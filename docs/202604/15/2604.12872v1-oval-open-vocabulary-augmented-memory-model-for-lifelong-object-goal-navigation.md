## 摘要
目标物体导航（Object Goal Navigation，ObjectNav）指的是智能体在未见过的环境中导航至目标物体的能力，这种能力常常是完成复杂任务所必需的。现有方法虽然在单一目标物体导航任务中表现出色，但其在终身记忆表征的适用性上存在局限，从而阻碍了智能体在较长时间内持续目标导航的有效性。为了解决这一问题，我们提出了 OVAL，一种新颖的终身开放词汇记忆框架，使得智能体能够在语义开放的任务中高效且精确地执行长期导航。在该框架中，我们引入了记忆描述符，以促进记忆模型的结构化管理。此外，我们提出了一种新颖的基于概率的探索策略，采用多值边界评分机制以提升终身探索的效率。大量实验结果表明了该系统的高效性与鲁棒性。

## Abstract
Object Goal Navigation (ObjectNav) refers to an agent navigating to an object in an unseen environment, which is an ability often required in the accomplishment of complex tasks. While existing methods demonstrate proficiency in isolated single object navigation, their limitations emerge in the restricted applicability of lifelong memory representations, which ultimately hinders effective navigation toward continual targets over extended periods. To address this problem, we propose OVAL, a novel lifelong open-vocabulary memory framework, which enables efficient and precise execution of long-term navigation in semantically open tasks. Within this framework, we introduce memory descriptors to facilitate structured management of the memory model. Additionally, we propose a novel probability-based exploration strategy, utilizing a multi-value frontier scoring to enhance lifelong exploration efficiency. Extensive experiments demonstrate the efficiency and robustness of the proposed system.

## 二、方法论与模型结构

### 2.1 核心思想  
OVAL 将“终身导航”建模为一个持续学习的问题，通过两个关键模块实现：
1. **结构化记忆模型** —— 通过记忆描述符（Memory Descriptor）形式化存储对象的语义、位置、场景信息；
2. **概率探索策略** —— 结合距离、语义相关性与探索足迹的多值前沿评分机制实现高效探索。

### 2.2 整体框架  
系统分为三大模块（Fig. 2）：
- **Frontier Exploration（前沿探索）**：根据深度与位姿生成网格地图，通过概率评分选择探索边界；
- **Open-Semantic Memory Model（开放语义记忆建模）**：利用 Grounded-SAM2 进行自动标注（Autolabel），经关键词过滤及同义类扩充后，生成记忆描述符；
- **Navigation（导航决策）**：通过 KMP 查询算法匹配目标词汇，在记忆中检索最优目标位置并执行导航路径规划。

### 2.3 技术细节与公式摘要  

- **概率探索公式**  
  \( P(F) ∝ o_d(F) + o_s(F) + o_f(F) \)  
  其中包括：
  - 距离因子 \(o_d(F)\)：优先选择距离较近的前沿；
  - 语义因子 \(o_s(F)\)：偏好与目标语义相关的房间区域；
  - 足迹因子 \(o_f(F)\)：避免重复探索已访问区域。

- **记忆模型结构**  
  每个记忆项为五元组：  
  \( S_i, I_i, X_i, H_i, C_i \)  
  其中分别代表：
  - \(S_i\)：语义标签；
  - \(I_i\)：实例图像缓存；
  - \(X_i\)：空间坐标描述符；
  - \(H_i\)：HSV 色彩空间的场景描述符；
  - \(C_i\)：置信度，用于目标确认。

- **实例匹配算法（Memory Management）**  
  利用 SuperGlue 特征匹配判断对象是否属于同一实例，依据置信度决定是否合并更新记忆。

- **停止验证机制（Verify “STOP”）**  
  当导航到候选目标后，系统通过 LLM 结合全景图像进行目标验证：  
  \( S = ωϕ(V_k) + (1 - ω)C_i \)，  
  若得分 \(S\) 超过阈值则触发停止，否则剔除低置信实例。
