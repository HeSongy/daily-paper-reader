## 摘要
我们提出了 Question-Asking Navigation（QAsk-Nav），这是首个用于协作实例目标导航（Collaborative Instance Object Navigation，CoIN）的可复现基准，能够对具身导航与协作式提问进行明确且独立的评估。CoIN 要求一个具身智能体在部分可观测的环境中，仅利用自我中心视觉观测与与人类的自然语言互动对话，根据自由形式的目标描述到达指定目标，其中对话可用于消除在视觉上相似目标实例间的歧义。现有的 CoIN 基准主要关注导航成功率，无法支持对协作交互的稳定评估。为解决这一不足，QAsk-Nav 提供了：(i) 一个轻量级的提问协议，其评分独立于导航；(ii) 一个增强的导航协议，具备真实、多样且高质量的目标描述；以及 (iii) 一个开源数据集，包含 28,000 条经过质量检查的推理与提问轨迹，用于 CoIN 模型的交互能力训练与分析。基于所提出的 QAsk-Nav 基准，我们开发了 Light-CoNav，这是一种轻量级的协作导航统一模型，其参数量比现有模块化方法缩小 3 倍、运行速度提升 70 倍，同时在对未见过的目标和环境的泛化能力上超过了最先进的 CoIN 方法。项目页面：https://benchmarking-interaction.github.io/

## Abstract
We propose Question-Asking Navigation (QAsk-Nav), the first reproducible benchmark for Collaborative Instance Object Navigation (CoIN) that enables an explicit, separate assessment of embodied navigation and collaborative question asking. CoIN tasks an embodied agent with reaching a target specified in free-form natural language under partial observability, using only egocentric visual observations and interactive natural-language dialogue with a human, where the dialogue can help to resolve ambiguity among visually similar object instances. Existing CoIN benchmarks are primarily focused on navigation success and offer no support for consistent evaluation of collaborative interaction. To address this limitation, QAsk-Nav provides (i) a lightweight question-asking protocol scored independently of navigation, (ii) an enhanced navigation protocol with realistic, diverse, high-quality target descriptions, and (iii) an open-source dataset, that includes 28,000 quality-checked reasoning and question-asking traces for training and analysis of interactive capabilities of CoIN models. Using the proposed QAsk-Nav benchmark, we develop Light-CoNav, a lightweight unified model for collaborative navigation that is 3x smaller and 70x faster than existing modular methods, while outperforming state-of-the-art CoIN approaches in generalization to unseen objects and environments. Project page at https://benchmarking-interaction.github.io/

## 一、研究问题与背景  

- **核心问题**：  
  现有协作实例目标导航（Collaborative Instance Object Navigation, CoIN）任务要求智能体在部分可观测环境中通过视觉和自然语言交互找到特定对象。然而，现有基准将“导航策略”与“人机交互质量”混合评估，导致：
  1. **不能独立衡量**模型在问答互动中的推理和交流能力；

  4. **任务描述质量差**：早期基于旧版VLM的目标描述语义不精确。  

- **研究动机**：  
  作者希望建立一个**轻量、可复现、可分解评估的基准体系**，显式分离“交互推理”和“导航策略”，支持更快速、公平的模型训练与测试，从而推动具身智能研究中人机协作的系统化分析。

## 二、方法论：QAsk-Nav 基准与 Light-CoNav 模型  

### 2.1 基准框架：QAsk-Nav  

整体由三个主要部分组成：  

1. **独立的问答协议（Question-Asking Protocol）**：  
   - 通过强化学习式过程模拟交互情境：智能体（questioner）根据目标描述 D 面对多个可能的物体图像 O，需决定是否提问或确认目标。  
   - 人类角色由“oracle”模拟，其拥有目标图像并给出回答。  
   - **输出结构**：`(D, O, R, S, Q, C)`  
     - D：目标描述  
     - O：当前视觉观测  
     - R：推理链条（reasoning trace）  
     - S：不确定性评分（0=非目标、1=不确定、2=确定目标）  
     - Q：如果S=1则生成问题；否则为空  
     - C：上下文中已有的问答对，支持多轮对话  
   - 允许多轮回合，直至智能体确认目标或出错终止。  

2. **增强的导航协议（Navigation Protocol）**：  
   - 改进自 CoIN-Bench，以更高质量描述驱动导航任务。  
   - 使用 **Gemini-3-pro** 等大型 VLM 生成精确目标文字描述，并由人工与 LLM 双重验证（80% 以上优选率）。  
   - 将任务划分为三种标准验证集：
     - **Val Seen**（相同对象但不同起点）  
     - **Val Seen-Synonyms**（相同物体类别，不同描述措辞）  
     - **Val Unseen**（全新环境与对象）  

3. **开放数据集 QAsk-Nav-dataset**：  
   - 共收录约 **28,000 条推理与问答样本**。  
   - 利用多模态生成模型（Gemini-3-pro，Qwen3-VL-30B-A3B 等）合成图像扰动、上下文多样性与多层级描述。  
   - 含6种描述等级（从仅“类别”到“颜色+上下文+显著特征”），用于分析描述信息密度对提问行为的影响。  

---

### 2.2 模型设计：Light-CoNav  

1. **核心思想**：  
   摒弃繁复的多模块管线（如AIUTA将VLM和LLM分离），整合为单一**轻量多模态Transformer模型**，实现统一问答推理与导航决策。  

2. **结构机制**：  
   - 采用基于 Qwen2.5-VL-7B 的轻量多模态模型（7B 参数）。  
   - 融合外部导航策略（如VLFM），仅负责交互决策。  
   - 输入 `(D, O, C)`，模型输出 `(R, S, Q)`。  
   - 不确定性驱动策略：
     - S=2 → 确定目标 → 执行接近操作；
     - S=0 → 放弃目标，继续探索；
     - S=1 → 生成问题 Q，与用户互动更新上下文。  

3. **训练流程（两阶段）**：  
   - **阶段一：推理与不确定性学习**  
     基于无提问样本的监督微调，学习生成充分解释的 R 与准确的 S 评分。  
   - **阶段二：提问能力微调**  
     继续在含提问数据 `(D, O, R, S, Q, C)` 上训练，比例约10:1混合，增强上下文关联与交互一致性。  

4. **效果优化**：  
   - 推理链生成显著提升模型的稳定性；
   - 多阶段训练使模型在问答识别与导航融合上更高效。

## 六、主要结论与发现  

- **QAsk-Nav基准**首次实现：
  - 问答与导航性能**解耦评估、可复现、低延迟**；
