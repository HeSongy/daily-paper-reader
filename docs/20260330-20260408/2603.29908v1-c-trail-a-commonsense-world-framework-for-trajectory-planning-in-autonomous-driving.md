## 摘要
自主驾驶的轨迹规划日益依赖大型语言模型（LLMs）来进行常识推理，然而LLM的输出本质上不够可靠，这在安全关键的应用中带来风险。我们提出了C-TRAIL，一个基于“常识世界”（Commonsense World）的框架，将LLM生成的常识与信任机制相结合，以引导轨迹规划。C-TRAIL通过一个闭环的“回忆（Recall）—规划（Plan）—更新（Update）”循环运行：回忆模块向LLM查询语义关系，并通过双重信任机制量化其可靠性；规划模块通过Dirichlet信任策略将加权的常识注入到蒙特卡洛树搜索（MCTS）中；更新模块根据环境反馈自适应地优化信任评分与策略参数。在Highway-env的四个仿真场景和两个真实世界levelXData数据集（highD, rounD）上的实验表明，C-TRAIL在各方面均优于当前最先进的基线方法，平均将ADE降低40.2%，FDE降低51.7%，并使SR提升16.9个百分点。源代码可在https://github.com/ZhihongCui/CTRAIL获取。

## Abstract
Trajectory planning for autonomous driving increasingly leverages large language models (LLMs) for commonsense reasoning, yet LLM outputs are inherently unreliable, posing risks in safety-critical applications. We propose C-TRAIL, a framework built on a Commonsense World that couples LLM-derived commonsense with a trust mechanism to guide trajectory planning. C-TRAIL operates through a closed-loop Recall, Plan, and Update cycle: the Recall module queries an LLM for semantic relations and quantifies their reliability via a dual-trust mechanism; the Plan module injects trust-weighted commonsense into Monte Carlo Tree Search (MCTS) through a Dirichlet trust policy; and the Update module adaptively refines trust scores and policy parameters from environmental feedback. Experiments on four simulated scenarios in Highway-env and two real-world levelXData datasets (highD, rounD) show that C-TRAIL consistently outperforms state-of-the-art baselines, reducing ADE by 40.2%, FDE by 51.7%, and improving SR by 16.9 percentage points on average. The source code is available at https://github.com/ZhihongCui/CTRAIL.

## 二、方法论概述

### 1. 框架总体思路

- **方法名称**：C-TRAIL（Commonsense-Trust Reinforced Autonomous Intelligent Learning）  
- **核心结构**：由三大模块组成的闭环循环——**Recall、Plan、Update**
  - **Recall**：从 LLM 提取结构化常识知识（如“前方有车”，“左侧可变道”），并通过**双重信任机制（Dual-Trust）**评估可靠性。  
  - **Plan**：基于信任加权的常识信息，构建 **Dirichlet 信任策略** 并注入蒙特卡洛树搜索（MCTS）引导决策。  
  - **Update**：根据环境反馈实时修正信任分数与策略参数，实现动态自校准。  

### 2. 核心技术细节

- **世界建模**：  
  定义“常识世界” \( W_t = (G_t, A, T) \)，其中 \(G_t\) 为信任加权的图表示，\(A\) 为动作空间，\(T\) 为状态转移函数。

- **Dual-Trust 机制**：
  - *语义信任* \( c_{t, llm} \)：评估 LLM 多次回答之间的一致性，用熵惩罚来抑制高不确定性输出；  
  - *运动学信任* \( c_{t, kin} \)：利用速度、加速度、航向、车道一致性等要素验证输出物理可行性；  
  - 二者综合为：  
    \[
    C_t = c_{t,llm} \cdot \sigma_\kappa(c_{t,kin} - \eta)
    \]
    若物理信任低，则抑制全部常识信任。

- **Dirichlet 信任策略**：  
  将 LLM 推荐的高层动作概率 \( f(a) \) 作为 Dirichlet 参数  
  \[
  \alpha_t(a) = f(a)(1 + \beta C_t)
  \]
  可信度高使策略集中，可信度低则平坦化，促进探索。

- **信任感知 MCTS 规划**：  
  修改 PUCT 选择规则：  
  \[
  a^* = \arg\max_a \left[ Q(h,a) + \lambda C_t \pi(a|Z_t,C_t)\frac{\sqrt{N(h)}}{1+N(h,a)} \right]
  \]
  动态平衡 LLM 引导与搜索探索。

- **信任校准更新（Trust Calibration Update）**：  
  执行后结合新观测与奖励更新信任值，衰减过时常识，增强奖励高的信任信息，使系统在运行中自我调节。
